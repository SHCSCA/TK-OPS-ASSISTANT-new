# v1.2.3 发布记录

发布日期：2026-03-29

## 变更摘要

- 修复素材中心进入页面时额外弹出 `python.exe` / 黑窗的问题。
- 移除素材中心首屏对视频海报的主动生成请求，避免页面加载链路触发后台进程。
- 将 `getAssetVideoPoster` 收口为纯缓存读取，缓存 miss 不再顺手调度后台生成。
- 保留导入素材、更新素材后的异步首帧海报补齐，但 Windows 下改为优先使用 `pythonw.exe` 并附带隐藏窗口启动参数。

## 根因

- 素材中心路由加载时，会对无海报视频主动请求视频海报。
- Bridge 在缓存 miss 时又会触发 `schedule_video_poster_generation`。
- 后台生成逻辑通过独立 Python 进程执行，在 Windows 环境下会暴露额外控制台窗口。

## 发布校验

- `node --check desktop_app/assets/js/page-loaders/asset-center-main.js`
- `venv\\Scripts\\python.exe -m py_compile desktop_app\\ui\\bridge.py desktop_app\\services\\asset_service.py`
- `venv\\Scripts\\python.exe -m pytest tests/test_page_loader_asset_split.py tests/test_page_runtime_data.py tests/test_bridge_runtime_contract.py tests/test_asset_video_poster_runtime.py tests/test_asset_service_video_poster_spawn.py -v`

## 版本治理

- 该版本应以 GitHub Release `v1.2.3` 作为客户端自动更新锚点

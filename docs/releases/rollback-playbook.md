# 发布回滚手册（Rollback Playbook）

## 适用场景

- 新版本安装后无法启动或关键主链路不可用。
- 运行时协议不兼容导致核心页面不可用。
- 线上反馈出现高频阻塞且无法在短时间内热修。

## 回滚目标

- 在最短时间恢复到“上一已验证可用版本”。
- 保证用户数据目录不被破坏。
- 对外有明确公告与后续修复路径。

## 回滚步骤

1. 确认回滚版本  
   - 选择最近一次完整通过门禁的版本号与提交号。
2. 产物准备  
   - 从已归档发布产物中取回上一版本安装包。
3. 执行回滚发布  
   - 更新发布说明，标记当前版本为已回滚。
   - 发布上一版本安装包并置顶回滚公告。
4. 验证回滚有效  
   - 执行安装后 smoke：首启、关键页面加载、核心操作。
5. 启动 hotfix 分支  
   - 基于回滚前版本定位根因，产出修复补丁与复盘。

## 用户沟通模板

- 标题：`[重要] TK-OPS vX.Y.Z 回滚通知`
- 内容最少包含：
  - 回滚原因（用户可理解）
  - 受影响范围
  - 建议操作（是否需要重新安装）
  - 下一次修复版本预计时间

## 复盘检查项

- 门禁是否遗漏了可提前发现该问题的检查项。
- 是否存在“文档已改、脚本未改”或“脚本已改、文档未改”。
- 是否需要新增测试覆盖本次故障类型。

# 旧壳字号回归 + 概览看板回归增强设计（真实数据版）

## 背景
当前新壳已具备基础壳层能力，但 dashboard 仍明显弱于旧壳：
- 字号在窄窗偏大，主标题与统计数字体感不一致。
- 概览结构不完整，缺少时间窗切换、趋势聚合、活动流联动、系统卡联动。
- 数据契约仅返回基础指标，无法支撑旧壳同等信息密度。

## 设计目标
1. 字号和排版基线回归旧壳。
2. dashboard 结构回归旧壳并补 2 个增强块。
3. 通过后端真实聚合补齐数据，不做前端造数。
4. 壳层状态与 dashboard 交互联动（右栏、底栏）。

## 方案总览
### A. Dashboard Runtime Contract
接口：`GET /dashboard/overview?range=today|7d|30d`

返回对象在保留现有字段的基础上新增：
- `trend: Array<{ label: string; created: number; completed: number; failed: number }>`
- `activity: Array<{ title: string; entity: string; category: string; status: string; time: string }>`
- `systems: Array<{ key: string; title: string; status: string; tone: 'success'|'warning'|'error'|'info'; summary: string }>`

兼容约束：
- 既有 `metrics/accountStatus/taskStatus/regions/recentTasks/activeProvider/settingsSummary` 保留。
- 无数据时返回空数组，外加可读空态说明文本，不填充伪业务数字。

### B. 后端聚合策略
- 时间窗：
  - `today`：当天零点到当前时间
  - `7d`：当前时刻往前 7 天
  - `30d`：当前时刻往前 30 天
- `trend`：使用 repository 的计数方法聚合 created/completed/failed。
- `activity`：使用 `list_recent_activity_logs(limit)`，映射实体、分类、状态、时间。
- `systems`：由 runtime 健康、任务状态、供应商接入、账户健康汇总生成，tone 按风险等级映射。

### C. 前端数据与状态
- `runtimeApi.getDashboardOverview(range?: 'today'|'7d'|'30d')`。
- `useDashboardData`：维护 `range`、`selectedActivity`、`selectedSystem`，支持切窗刷新。
- `useShellStore` 新增：
  - `dashboardRange`
  - `selectedActivity`
  - `selectedSystem`
- 当用户选中活动行/系统卡时：
  - 更新右栏摘要（通过 `setRouteSummary`）
  - 更新底栏左侧状态文案（当前焦点）

### D. Dashboard 页面结构
- 顶部：`today/7d/30d` + `查看历史` + `刷新数据` + `新建任务`。
- 第一屏：4 KPI 卡（含异常任务跳转）。
- 左列：趋势模块 + 活动流表格（行可选中）。
- 右列：系统状态卡（可选中）+ 快捷入口/待办 + 异常聚焦 + 今日待办。
- 空态统一：显示“暂无数据/等待数据”并提供恢复动作（刷新）。

### E. 字号与排版
- 壳层统一字号令牌：
  - `--shell-font-body: 13px`
  - `--shell-font-subtle: 12px`
  - `--shell-font-h1: 32px`
  - `--shell-font-h1-compact: 20px`
  - `--shell-font-stat: 30px`
  - `--shell-font-stat-compact: 22px`
- 断点：`<=1100`/`<=960` 回落到 compact 级字号。
- 关键值使用：`white-space: nowrap; word-break: normal; overflow-wrap: normal;` 防逐字断裂。

## 风险与处理
1. 数据不足导致图表为空：
- 处理：明确空态，不伪造曲线。

2. 旧字段兼容回归风险：
- 处理：只增不删，测试覆盖旧字段与新增字段。

3. 窄屏可读性与信息密度冲突：
- 处理：优先结构稳定，次要信息下放到详情联动与状态面板。

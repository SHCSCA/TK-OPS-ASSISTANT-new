# 概览数据看板重构设计

## 目标

将 `dashboard` 从半静态展示页重构为真实可操作的总览页，解决以下问题：

1. 页头 `新建任务` 无动作
2. `今日 / 近7天 / 近30天` 只有选中态，没有真实筛选逻辑
3. 趋势图、活动流、AI 引擎状态存在大量静态或伪数据
4. AI 任务趋势图信息表达不清，缺少坐标含义
5. AI 引擎状态区选中样式与模块语义不匹配

## 范围

只重构 `dashboard` 页面自身，不扩散到其他页面。

涉及文件预计包括：

- `desktop_app/assets/app_shell.html`
- `desktop_app/assets/js/page-loaders.js`
- `desktop_app/assets/js/bindings.js`
- `desktop_app/assets/js/routes.js`
- `desktop_app/assets/css/components.css`
- `desktop_app/assets/css/interactions.css`
- `desktop_app/assets/css/shell.css`
- `desktop_app/ui/bridge.py`
- `desktop_app/database/repository.py`
- `tests/` 中与 dashboard 真实性、交互、页面审计相关测试

## 设计原则

- 不再保留“能看不能用”的按钮与筛选器
- 不继续展示无法从真实数据支撑的业务指标
- 能后端聚合的逻辑尽量放后端，不在前端拼伪值
- 前端只负责渲染、切换时间范围、展示明确的图例与说明
- detail panel 展示当前区块对应的真实摘要，不保留无关模板文案

## 页面结构调整

### 1. 页头动作区

- 保留：`今日 / 近7天 / 近30天`
- 保留：`查看历史`
- 保留：`新建任务`

行为定义：

- `新建任务`：直接打开任务表单
- `查看历史`：跳转 `task-queue`
- 时间范围按钮：切换 dashboard 当前时间窗口，并触发整页重算

### 2. 核心指标区

保留四张卡片，但全部改成真实可解释指标，不再混入伪 ROI：

- 账号总量
- 任务总量
- 异常任务数
- AI 供应商接入数 / 分组数（二选一，按可解释性落位）

如果没有真实同比/环比数据，则卡片副文案改成状态说明，不伪造百分比涨跌。

### 3. 趋势图区

移除静态 SVG 曲线，改成基于真实聚合结果渲染的简单趋势图。

图表约束：

- X 轴含义明确：时间桶
  - 今日：按小时
  - 近7天：按天
  - 近30天：按天或周（按当前数据密度决定）
- Y 轴含义明确：任务数
- 系列只保留真实可支撑的 2-3 条：
  - 新增任务
  - 已完成任务
  - 异常任务

标题与图例必须明确写出单位，不再使用“完成率”“异常占比”这类当前未稳定建模指标。

### 4. 近期活动流

移除当前写死的 3 行模板记录，改为真实活动源：

- 优先使用 `activity_logs`
- 如果活动日志信息不足，则用近期 `tasks` 作为回退

展示字段：

- 动作名称
- 关联对象
- 分类
- 状态
- 时间

detail panel 展示对应行的真实详情。

### 5. AI / 系统状态区

移除伪服务名，如 `Video Engine v4.2`。

改成系统级摘要项，例如：

- 任务执行状态
- AI 供应商接入状态
- 账号/同步状态
- 下载或设备链路状态

每项状态必须能映射到真实数据来源。

交互上保留可选中能力，但使用 dashboard 专属轻量选中样式，不复用通用 `.task-item.is-selected` 粗 outline 风格。

## 数据流设计

### 后端

补一个 dashboard 专用聚合返回结构，至少覆盖：

- 顶部指标
- 时间范围趋势数据
- 近期活动流
- 系统状态摘要

建议沿用 bridge JSON 信封规范，在 `bridge.py` 中新增或扩展 dashboard 聚合接口，在 `repository.py` 中补充所需查询。

### 前端

- `page-loaders.js` 负责：
  - 拉取 dashboard 聚合数据
  - 根据当前时间范围刷新页面
  - 渲染卡片、趋势图、活动流、状态列表
- `bindings.js` 负责：
  - 页头按钮动作
  - 时间范围切换事件
  - 状态项选中态和 detail panel 更新

## 错误处理

- 后端聚合失败时：
  - 显示明确空态或失败态
  - 不保留旧静态假数据兜底
- 某个子区块无数据时：
  - 用诚实空态文案
  - 不伪造趋势线或活动记录

## 测试策略

至少补以下验证：

1. dashboard 页头 `新建任务` 有真实动作
2. `查看历史` 仍跳 `task-queue`
3. 时间范围切换会触发 dashboard 重新渲染
4. dashboard 页面不再依赖静态活动流记录
5. dashboard 页面不再保留伪 AI 引擎服务名
6. 趋势图区域文案明确标注坐标/单位语义

## 风险与控制

- 风险：dashboard 现在是模板直写，重构时容易和现有 detail / 交互绑定打架
- 控制：仅替换 dashboard 自身模板、loader、bindings，不顺带改其他 route
- 风险：现有数据库里时间字段分布不稳定，趋势桶可能为空
- 控制：允许空态，但不允许回退成伪数据

# 账号管理生产预备版设计

## 背景

新桌面壳与新 Python runtime 已经建立主链路，但当前账号管理仍停留在基础 CRUD 与单次连通性测试层级。它缺少生产环境常用的双状态模型、批量操作、归档语义、检测回写边界与审计摘要，无法支撑高频运营与稳定验收。

## 目标

把新架构下的账号管理收口成一个可持续扩展的垂类模块，首轮达到“生产预备”水平：

- 同时展示并维护人工状态与系统状态
- 支持高频筛选、批量操作与归档优先
- 系统检测只回写系统状态，不覆盖人工状态
- 在列表与详情视图中暴露最近错误、最近检测、绑定环境与基础审计摘要
- 保持前后端契约清晰，避免旧壳状态字段语义污染新 runtime

## 非目标

- 本轮不实现完整 CSV/Excel 导入向导
- 本轮不实现复杂权限系统
- 本轮不把账号检测完全任务化为独立调度中心
- 本轮不重写旧 `desktop_app` 的账号页面交互

## 设计原则

### 1. 双轨状态

账号主视图必须同时展示：

- 人工状态：运营口径，沿用账号主状态字段
- 系统状态：来自真实检测链路
- 风险状态：独立于人工状态和系统状态

人工状态与系统状态不互相覆盖。系统检测失败只能改变系统状态、检测时间和错误摘要，不能自动把人工状态改成暂停或禁用。

### 2. 归档优先

生产预备阶段默认优先归档，不鼓励直接删除。删除保留现有接口以兼容已有链路，但新 UI 以归档作为主退出动作。

### 3. 列表工作台优先

账号管理以列表工作台为核心，详情侧板为补充，不走复杂多级页面跳转。运营常用动作应在列表页完成：

- 搜索
- 状态筛选
- 批量选择
- 批量改人工状态
- 批量分组
- 批量归档
- 批量发起系统检测

### 4. 审计摘要优先于完整审计页

本轮不单独做完整审计模块，但账号详情必须能看到最近几条与该账号相关的活动摘要，为后续扩展完整审计中心预留数据结构。

## 数据模型

在现有 `Account` 基础上新增或明确以下语义：

- `status`：人工状态
- `last_connection_status` / `last_login_check_status`：系统状态来源
- `risk_status`：风险状态
- `archived_at`：归档时间
- `archived_reason`：归档原因

推荐的系统状态呈现规则：

- 优先显示 `last_login_check_status`
- 若登录态未校验，则回退到 `last_connection_status`
- 若两者都无有效值，则显示 `unknown`

风险状态首轮使用枚举：

- `normal`
- `watch`
- `high_risk`
- `frozen`

## Runtime API 设计

### 读取

- `GET /accounts`
  - 支持 `query`
  - 支持 `manual_status`
  - 支持 `system_status`
  - 支持 `risk_status`
  - 支持 `include_archived`

返回：

- 列表项
- 总数
- 已筛选总数
- 基础统计摘要

### 单条写入

- `POST /accounts`
- `PUT /accounts/{id}`
- `POST /accounts/{id}/archive`
- `POST /accounts/{id}/unarchive`
- `POST /accounts/{id}/test`

### 批量写入

- `POST /accounts/bulk`

动作首轮支持：

- `set_manual_status`
- `set_risk_status`
- `assign_group`
- `archive`
- `unarchive`
- `test_connection`

## 前端页面设计

### 列表页

主列表显示：

- 账号名 / 平台 / 地区
- 人工状态
- 系统状态
- 风险状态
- 分组
- 绑定设备
- 最近检测时间
- 最近错误摘要

列表页顶部提供：

- 关键词搜索
- 人工状态筛选
- 系统状态筛选
- 风险状态筛选
- 是否显示归档

### 批量操作条

只有勾选账号后显示，提供：

- 批量改人工状态
- 批量改风险状态
- 批量分组
- 批量检测
- 批量归档 / 取消归档

### 详情侧板

展示：

- 基础资料
- 双状态与风险状态
- 绑定环境信息
- 最近错误
- 最近活动摘要

## 审计设计

复用 `ActivityLog`，以 `related_entity_type='account'` 与 `related_entity_id=<account_id>` 记录账号活动。首轮记录：

- 创建账号
- 编辑账号
- 批量更新
- 归档 / 取消归档
- 发起检测
- 检测成功 / 失败

## 测试策略

至少补齐以下测试：

- runtime 契约测试：筛选、批量操作、归档、检测回写
- frontend 合约测试：双状态字段、批量操作条、详情摘要
- vertical slice：创建账号 -> 批量改状态 -> 发起检测 -> 系统状态回写但人工状态保持
- 数据库生命周期测试：新增账号字段迁移后 head 正常

## 本轮实现范围

本轮只实现“生产预备版第一阶段”：

- 双状态显示
- 风险状态
- 归档语义
- 列表筛选
- 批量操作
- 审计摘要
- 对应契约与垂类测试

导入向导和完整审计中心留待下一阶段。

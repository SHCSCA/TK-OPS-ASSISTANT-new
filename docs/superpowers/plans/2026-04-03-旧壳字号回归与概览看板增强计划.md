# 旧壳字号回归 + 概览看板回归增强执行计划（真实数据版）

## 1. 目标
- 让新桌面壳在 `<=1100/960` 的字号体感回到旧壳：`h1 32/20`、`stat 30/22`、导航正文 `13`、辅助文案 `12`。
- 让概览看板恢复旧壳核心结构，并在不造数前提下补齐增强块（异常聚焦、今日待办）。
- 后端 `dashboard overview` 改为可按时间窗返回真实聚合数据，前端交互与右栏/底栏状态联动。

## 2. 范围
- 前端：`apps/desktop`（dashboard 页面、shell store、runtime types/api、main.css）。
- 后端：`apps/py-runtime` legacy facade + dashboard 路由。
- 测试：`tests/test_runtime_api.py` + 必要的前端静态断言测试。
- 文档：`docs/superpowers/plans`、`docs/superpowers/specs`。

不在本轮范围：
- 新增独立页面。
- 非 dashboard 页面的深度视觉重构。
- 640px 以下完整布局优化（只保证不崩不重叠）。

## 4. 分阶段执行
### 阶段 A：测试先行（TDD Red）
- 在 `tests/test_runtime_api.py` 增加 `dashboard overview range + trend/activity/systems` 断言。
- 增加空数据分支断言：返回空数组 + 空态说明，不造假数据。

### 阶段 B：后端契约扩展（TDD Green）
- `GET /dashboard/overview` 支持 `range=today|7d|30d`。
- facade 新增时间窗聚合：
  - `trend[]`：created/completed/failed 汇总
  - `activity[]`：最近活动流
  - `systems[]`：runtime/任务/供应商/账户摘要
- 保持已有字段兼容，不破坏旧调用方。

### 阶段 C：前端数据层与状态联动
- runtime API 增加 `getDashboardOverview(range)`。
- dashboard composable 增加 range 状态与 reload。
- shell store 增加：`dashboardRange`、`selectedActivity`、`selectedSystem`。
- 右栏摘要与底栏 chip 支持 dashboard 交互联动更新。

### 阶段 D：概览页结构回归 + 增强
- 恢复旧壳结构：时间段切换、4 KPI、趋势、活动流、系统状态、快捷入口。
- 增强块：`异常聚焦`、`今日待办`，仅使用后端真实统计/列表推导。
- KPI 卡支持跳转：异常任务卡直达任务队列异常视图。

### 阶段 E：字号与排版回归
- 在 `main.css` 统一字号令牌与断点：`h1 32/20`、`stat 30/22`、正文13、辅助12。
- 关键值（IP/端口/版本/数字）使用不可逐字断裂策略。

### 阶段 F：回归验证
- `npm run typecheck`
- `npm run build`
- `venv\Scripts\python.exe -m pytest tests/test_runtime_api.py -v`
- `venv\Scripts\python.exe -m pytest tests/test_desktop_frontend_routes.py -v`（若改动触发）
- `npm run tauri:smoke`

## 5. 边界与回退点
- 若 dashboard 新契约导致旧字段不兼容，立即回退到“新增字段不替换旧字段”的最小策略。
- 若窄屏下仍出现挤压，优先保证结构稳定（隐藏次要信息），不为信息完整牺牲布局。
- 若真实数据缺失，展示空态文案，不引入模拟业务值。

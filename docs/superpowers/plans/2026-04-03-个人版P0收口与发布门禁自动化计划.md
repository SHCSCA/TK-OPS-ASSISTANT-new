# 个人版 P0 收口与发布门禁自动化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在不改动产品定位的前提下，把个人版 P0 主链路推进到“可持续验证、可重复发布”的状态，并把发布前门禁从人工清单升级为可执行脚本。  
**Architecture:** 以 `apps/desktop + apps/py-runtime` 为唯一主链路，围绕“协议一致性、账号域闭环、稳定性、发布门禁”四条线并行推进；优先补齐自动化验证入口，确保每个能力改动都能被门禁捕获。  
**Tech Stack:** Python 3.11, pytest, FastAPI, SQLAlchemy, Vue3/TypeScript, PowerShell, Tauri

## 4. 验证方式

- 每阶段至少执行：
  - 改动相关 pytest
  - 一条主链路 smoke
  - 文档同步检查（README + docs/releases）
- S4 完成判定：
  - `scripts/preflight-gate.ps1 -Full` 全绿
  - 目标机安装后首启与关键主链路通过
  - 回滚路径文档可按步骤演练

## 5. 回退点

- S1 回退点：仅移除新增门禁脚本与调用，不影响现有 build/release。
- S2/S3 回退点：按账号域接口粒度回退，不跨域回退 runtime 主干。
- S4 回退点：保持上一可发布版本，单独回滚 release/smoke 变更并发布 hotfix 说明。

# TK-OPS 重构执行排期表（第 1 周 ~ 第 4 周）

## 适用前提

本排期基于当前仓库真实状态制定，不是从零开始：

- 当前项目已经是 **PySide6 + QtWebEngine + QWebChannel + SQLite + SQLAlchemy** 的本地桌面方案。
- Python 侧已经存在 service、repository、sample db、测试与 seed。
- 当前前端仍是 **HTML / CSS / JavaScript**，页面运行时由大型 `routes.js` 与 `page-loaders.js` 驱动。
- 目标架构是：**Tauri + Vue 3 + TypeScript + Python Sidecar Runtime + SQLite**。
- 重构策略是：**保留 Python 业务资产，替换桌面壳与前端表示层，渐进迁移页面**。
